纯 CSS 实现视差滚动

发表于 2016-08-01

再说 CSS 简单我打你哦 （白眼

之前看到一篇博客，想要用 CSS 尝试一下实现视差滚动效果，本来以为只是这几个关键属性就可以了：

 transform-style: preserve-3d;
 perspective: 1px;
 transform: translate3d(0, 0, -1px) scale(2);

然而试了之后，

你倒是滚啊。。。

研究之后发现其实很多细节需要好好理解，首先原理是这样的：视差滚动本来就是一种天然的透视现象，不仅有“近大远小”，在“场景”发生移动的时候，远处的物体比距离近的物体移动距离也要小，所以利用 CSS 中的 perspective 属性让整个场景形成透视就自然形成视差滚动，而 3D 的 z 轴偏移值在 css 中就是 translate3d属性中 z 值，此时 也不需要设置 transform-style: preserve-3d;

既然要让“场景”发生滚动，就要让具有透视属性的“场景元素”出现滚动条，所以第二个必要条件：

  • overflow：auto/scroll (或者直接使用 overflow-x: hidden，顺便避免水平滚动条)；
  • 场景内元素的高度要大于场景元素的高度；

在实际效果里我们往往不需要“近大远小”，用 scale 处理一下，对较远处的物体进行放大来抵消这个效果，可以用对应的公式：1 + (translateZ * -1) / perspective

这里可以看到三个元素左侧有空隙，并不是 scale 放大的数值不够，它的宽度其实已经恢复到所在包含块的宽度值 ，由于包含块滚动条的出现父元素总宽度变大了一点，因为水平方向也有透视效果，所以三个元素虽然宽度没有变，但是会被水平放置在整个父元素宽度的中心

还有一个问题，如果要应用这个效果的话，因为场景元素必须有滚动条，所以设置其他元素的时候要注意不能再出现滚动，尤其是视口容易出现滚动条的嵌套；

关于这个效果的实际应用还有很多细节，以及需要更好的理解 3D 效果中的透视问题，下次补充~
