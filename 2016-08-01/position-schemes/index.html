Position Schemes 之 Absolute Positioning

发表于 2016-08-01

position schemes

display 属性决定了盒子的类型，而 position 和 float 则确定它们该如何进行定位 -- 分为三种基础的 position schemes：

  • normal flow
  • floats
  • absolute positioning

flow of an element

我们认为一个文档的根元素root element是 out-of-flow 的，它和它内部的 in-flow 元素构成属于根元素的一个流 （flow），而其中如果有一个元素产生浮动或者是 absoluteiy positioned 时，我们认为这个元素也会out of flow，它就又和它内部的 in-flow 元素形成一个新的流。

box offsets

已定位的元素 positioned element 就会拥有四个偏移值属性 top right bottom left ，它们指定了自身 box 的 margin edge 的偏移值，不过偏移的参照对象在不同定位中会不一样，初始值默认为 auto； 它们的百分比数值和 width margin padding 这些属性一样也是相对于包含块，不过既然是表示已定位的元素的偏移值， 垂直方向的 top bottom 就相对于包含块的高度，水平方向相对于包含块宽度，偏移量也可以是一个负值； 比如 position： absolute 元素的包含块是最近的已定位元素的 padding edge，所以 top、 right 、width等这些属性的百分比数值的计算，都是相对于 这个 padding box 的大小。

absolute positioning

我们说元素或者它的 box 是 absolutely positioned 指的是包含了 position： absolute / fixed 两种情况，它们唯一的区别就是包含块的不同； absolute positioning 中的元素完全从 normal 中脱离出来，不会对后面的兄弟元素产生布局上的影响，也不会 flow around 其它元素，只会根据 stack levels 发生重叠；因为属于 out of flow 的元素，因此它们的 margin 永远不会 collapsed ； absolutely positioned 盒子的偏移值参考于它们包含块；也就是说位置的计算从包含块的四个边界开始。

位置和大小的计算

在这一节讨论时都没有涉及 padding border 的宽度，因为它俩的取值没有 auto 这种情况，宽度要么是设定的确定值，要么是默认的 0，所以不会影响位置的分类讨论

1. 水平方向

首先无论什么时候都是满足这个等式的： left + margin-left + 'border-left-width' + 'padding-left' + width + padding-right + border-right-width + margin-right + right = width of containing block

静态位置

当 left 和 right 都是 auto 时，相当于没有指定水平方向上相对于包含块的位置，这时候它的位置参考于在 normal flow 中的一个位置 static position：

我们设定一个假想盒 a hypothetical box ，即这个元素在 position：absolute 并且 float：none 的情况下产生的第一个盒子，注意这时候可能 display 属性的计算值也受到了影响，可能会恢复到他的 设定值，用这个假想盒所在的包含块作为 static-position containing block ； 把所有可以滚动的盒子滚动到起点位置，（所以如果是 fixed 定位的元素，包含块是 initial containing block 而不是 viewport）得到它现在所在的位置，以它的 left margin edge 到包含块左边界的距离为 static position for 'left' ，它的 right margin edge 到包含块左边界的距离为 static position for 'right'。

rather than actually calculating the dimensions of that hypothetical box, user agents are free to make a guess at its probable position.

终于得到了静态位置之后。。。margin 值为 auto 的话就置为 0 ，（因为我们用偏移值来满足上面那个公式就可以了，除非是下面的一种特殊情况） 如果left 和 right 都是 auto 值，就把它的 left 属性设置为 static position 的 left 值，（这里如果 static-position containing block 所在的元素的 direction 属性是 rtl ，就设置 right 为 static position for 'right' ） 如果left 和 right 设置了其中之一，就使用设定值 然后进行width 宽度的判断，用它的设定值，或者是auto 的时候就 shrink-to-fit 最后把right 属性自动算出保证公式成立就可以了（这里如果 static-position containing block 所在的元素的 direction 属性是 rtl ，就先设置 static position for 'right' ，得出width，再计算 left）。

如果 left 和 right 都设定了值而 width 是 auto ，那就计算宽度值去满足公式；

当 left 、margin 、right 三个值都设定了非 auto 的值时： 此时如果 margin-left 和 margin-right 同时为 auto ，则在计算公式的时候保证它们相等，但是如果这时他们的计算值是两个负数，就让 margin-left 为 0，再直接算 margin-right（ direction 属性是 rtl ，就设置 right 为 0 再算左边）； 如果 margin-left 和 margin-right 也设定了非 auto 的值，然后整体 over-constrained ，即无法满足上面的公式的时候，就改变偏移值使公式成立： 忽略 right 的设定值直接用公式计算 （同样，如果 direction 是 rtl ，就忽略 right 的设定值）。

shrink-to-fit

这种计算宽度值的算法中定义了三种宽度：

  • preferred width ：首选宽度，除了有明确换行的地方，把内容全部以不换行做处理得出的宽度；
  • preferred minimum width ：首选最小宽度，尽可能的将内容换行后得到的宽度；
  • available width ：可用宽度，结合上面的具体使用情况来看，就是把此时计算值还没有确定的 left 或者 right 置 0，然后就只剩 width 没有明确大小，根据公式算出来的值就是可用宽度。 根据这个公式判断，上面这三种宽度选择哪一个： min(max(preferred minimum width , available width), preferred width).
