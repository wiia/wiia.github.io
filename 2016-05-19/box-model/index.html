Box Model

发表于 2016-05-19

在 CSS 中，元素通过生成 box 表现在视觉格式化模型中。

简单的说，在一个盒子中：
conent area - content edge/inner edge - content box
padding area - padding edge - padding box
border area - border edge - border box
margin area - margin edge/outer edge - margin box（只有它的 background 是透明的）

margin：<length> | <percentage> | auto | inherit
margin：<length> | <percentage> | inherit

初始值都为0；
它们的百分数值，包括垂直方向上的，都是以 containing block 的width值为基准；
margin 的<length>值可以为负数；
它们在一些 table 元素中会失效，其中margin不作用于非替换inline元素的垂直方向。

Collapsing margins

多个 相邻的 垂直margin会折叠。

那么，什么情况会认为 margins 之间是 adjoining 的呢？
首先这几个 margins 所属的 boxes 都要满足：

  • in-flow；
  • 是在同一个 BFC 中的 block-level box；

所以有很多时候都是不会发生折叠的：
floated box ，absolutely positioned box以及根元素都会 out-of-flow，还有 inline-block box 这些不属于 block-level box 的，
它们的 margin 不会和任何其他 margin 发生折叠，包括和它内部的 in-flow 子元素；
还有其他的会产生新的 BFC 的情况，当元素overflow值不是visible时，内部的子元素和它已经不属于同一个 BFC ，所以也不会 collapse。

  • 然后当然要满足的是，margin要属于垂直相邻vertically-adjacent的 box edges，意味着中间没有 line boxes，clearance，垂直方向的padding，border这些东西相隔。

除了上下相邻的两个 box 之间的 bottom、top margin 以外，
第一个子元素和父元素的 top margin；
最后一个子元素和 height为auto的父元素的 bottom margin；

或者如果一个元素的 box 里面什么也没有（没有 line boxes，clearance，垂直方向的padding，border，而且min-height：0 ， height：0/auto,不过可以包含in-flow子元素的collapsed margin）的时候，它自己的 bottom、top margin；
都是垂直相邻的。

总的来说，在一个flow中，block-level box margin的作用是为了保障它的 border box 在当前的 BFC 中，周围有一定的区域内是没有其它元素 border box 的，所以两个margin相邻时，只要取一个最大值就可以保证相同的效果。
