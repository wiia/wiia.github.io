视觉格式化模型 visual formatting model

视觉格式化模型关系到浏览器如何在屏幕上渲染dom树。文档中的每个元素会根据box model生成0到多个boxes，而这些盒子的布局决定因素有：

  • 盒子的尺寸和类型；
  • 定位方案 positioning scheme：normal flow, float, and absolute positioning；也就是说，display最终的计算值还要参考position和float属性。
  • dom树中元素之间的关系；
  • viewport大小，图片尺寸等；
视觉模型对 continuous media 和 paged media都适用，规范中没有对渲染规则面面俱到，所以一些细节在不同浏览器中表现是不一样的。

viewport和包含块containing block

在continuous media中

用户通过浏览器提供的viewport来查看页面，viewport是根元素html的包含块initial containing block ，所以viewport大小影响着文档的布局。 当viewport小于渲染文档的canvas，浏览器就会出现滚动条。

一个盒子（box）为它的后代建立了包含块containing block，也就是说包含块并不指 principal box（见下文）本身 。

每个盒子的位置和大小参考于它的包含块，但并不被它所限制，可能会overflow.

  • position： fixed：包含块和<html>一样由viewport生成
  • position： position：

    Box's types

在视觉格式化模型中，display会声明盒子的类型，影响它在其中的表现。一旦确定了display属性的值，元素会产生一个principal box，在里面包含后代boxes，产生context，同时也是这个盒子参与到positioning scheme中。

一些元素还会产生额外的盒子additional boxes,这些框的位置会参考于principal box display:list-item会使<li>另外给项目符号生成一个marker box

Block boxes

我们通常意义讲的,当 一个元素的principal box为block box时，它有两个身份：

  • display：block/ list-item /table：作为block-level box,参与到BFC中，产生它的元素称为Block-level element。
  • display：block /list-item/ inline-block（非替换元素）：它又同时是一个block container box，产生它的元素又叫block container element。

同时为block-level box和block container box的盒子称为block box，所以其实关系就是： 盗图。

Inline boxes

同样，inline boxes也会区分它在父级上下文当中的角色，以及盒子内部对后代的影响。

  • display：inline /inline-table /inline-block：Inline-level element -> inline-level box ，参与IFC。 但是一些元素的盒子内部没有生成IFC，比如虽然是inline-level但是是替换元素的<img>； 以及 display：inline-block的元素，虽然生成的是inline-level box，但是盒子内部会格式化为一个block box，（所以它生成的是一个principal inline-level block container）

    They are called atomic inline-level boxes because they participate in their inline formatting context as a single opaque box.

  • display：inline（非替换元素）：Inline element -> inline box 此时要求它的content也要在IFC中。

Anonymous boxes

（匿名框的属性往往无法设置， inheritable properties会继承自外层的非匿名框，计算百分比值的时候也是这样，不会参照于匿名框；非继承的属性就保持它的初始值） 上面提到的block container box，在这个盒子中有两种情况：

  • 只包含block-level boxes。
  • 产生IFC，即只包含inline-level boxes。

而匿名框的产生就可以避免BFC和IFC同时出现 比如出现了这种情况： 在一个block container box里同时出现了inline content 和 block content

<div>
  Some text
  <p>More text</p>
</div>

只要有一个block-level box包含在里面，就会强制它符合第一种情况，所以要给“Some text”加上一个anonymous block box

或者还有一种类似的情况，在一个inline box里出现了block-level box （我实在想不出什么情况下会这样用，不过没关系我们先讲道理）

p { display: inline }
span { display: block }

<p>
  This is anonymous text before the SPAN.
  <span>This is the content of SPAN.</span>
  This is anonymous text after the SPAN.
</p>

所以这个inline box （以及同一个行框中的祖先inline boxes，如果有的话 ）会被里面的若干个连续的block-level boxes 分成两个anonymous block box。这样，他们三个就形成了兄弟关系。

（当然，这种情况下inline box上如果设置了属性值，也可以控制匿名框的属性，比如例子中的<p>元素）

更常见的情况是文本直接出现在block container box中：

<p>Some <em>emphasized</em> text</p>

这时候<p>的block container box就属于产生IFC的第二种情况。因为"Some"和"text"不在inline element中，所以<p>元素就给它们的生成了两个anonymous inline boxes

另外： display：none使得元素以及它的子元素完全从formatting structure 中移除，不会产生任何boxes，也不会影响布局；即便是子元素中再次声明其他的display值也不会覆盖。
