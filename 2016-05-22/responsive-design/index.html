移动端设备的响应式初步

发表于 2016-05-22

基础概念

物理像素（physical pixel）是屏幕的最小物理显示单元，也就是屏幕分辨率中计算的像素点。
而我们在网页设计中设置的尺寸(css像素)则属于设备独立像素（device independent pixels)（或者dip/dips/密度无关像素…都是指的它)，用来表示逻辑像素中的一个“点”。

在高清设备中这两者往往是不相等的，比如一个1px1px的点可能由22个物理像素表示，我们用dpr（devicePixelRatio）来表示这个比例：
window.devicePixelRatio = physical pixels / dips（同一x/y方向上）

dpr是一个设备默认的值，往往是物理像素与设备宽度screen.width的比值

The Window.devicePixelRatio read-only property是一个只读属性
There are no callbacks or events to listen to when this value changes.没有回调函数或者是事件来监听这个值的改变

viewport最初由苹果开发者提出，用来解决desktop页面在移动端打开的适应问题。
桌面浏览器的viewport就是我们可通过拖动浏览器窗口来改变大小的可视区域visible area，而在移动设备中就与visible area大小无关，而是定义了页面内容的包含区，js中通过document.documentElement.clientWidth获得。

ios中viewport默认宽度是980px（高度会根据“宽度”以及“设备的宽高比”按比例设定），而浏览器会认为你的页面会在可视区域内全部展示出来，所以 scale的默认值会使页面缩放到宽度为device width。
比如，当我们只设定一个属性content="initial-scale=1.0"，设备宽度为320px，它会自动推算viewportwidth为320px。

参考Apple documentation:Configuring the Viewport

响应式方案

当我们页面中某些尺寸需要根据屏幕大小而改变时，当然一种做法是media query。

如果需要尺寸数值更精确地响应屏幕大小变化，可以使用rem单位，让数据根据html根节点的font-size大小而改变。
然后就要根据viewport大小来设置font-size：
例如一份750×1334的视觉稿：
font-size = 75 px （750 px / 10）
视觉稿中270px的尺寸则设置为270 px / 75 = 3.6 rem
当然这个过程会放到less等预编译工具里进行转换。
